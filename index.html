Wordpress Help Class

You love Wordpress and love your clients. You've built perfectly customized custom post types and metaboxes for them. Yet even though it should be obvious how it works, someone always comes around sending an email asking. How do we do this? This class can help.

With two simple lines of code you can take advantage of the rich editing capabilities already built into Wordpress. Add images using the media library, use lists, heading, etc. And it's easy to update. No pushing code to the server when someone asks for their documentation to be updated. Just login to the admin interface, make the change, and click publish.

Benefits

  • Quickly and easily add and update help to your Wordpress theme or plugin.
  • Uses the same Editor as all other Wordpress posts
  • Uses the media library to store images
  • Update your documentation using the admin interface and not by pushing code to your production server
  • Easily add contextual help or a help metabox or even both to your edit post screen for posts, pages, or custom post types

Drawbacks

  • Adds additional databases calls to admin requests
  • Without appropriate permission structure users could potentially change documentation
  • Changes do not easily sync back to development sites
  • Not a plugin

How It Works

Install the class in your theme or plugin directory. 

git clone https://sloped@github.com/sloped/Wordpress-Help.git

Include the class file and create your first help page

include 'wp-help-class/help-class.php';
$help = new wp_help();
$help->create_help(array('post_type' => 'post','page_title' => 'Blog Post Help'));

Now login to your admin interface and click on the new Help custom post type

Click on the new Blog Post Help page.

Edit as you wish

Click Update and either edit or create a new post. You will see the text you entered in the Blog Post Help post appear in both contextual help and as a metabox on your screen.
